## 1. 導入準備

> はじめてこのツールを使う方向けのセットアップ手順です。セットアップ済みの方は [2. アプリの概要](#2-アプリの概要) へ進んでください。

### 1-1. Homebrew のインストール

**Homebrew** は macOS 用のパッケージ管理ツールです。Python や Git などのソフトウェアをターミナルから簡単にインストールできます。

**インストール**（公式サイトのコマンドを1行コピーして実行）：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

> ターミナルを開くには：Finder → アプリケーション → ユーティリティ → ターミナル

**インストール確認**：

```bash
brew --version
```

`Homebrew 4.x.x` のように表示されれば成功です。

---

### 1-2. 必要なアプリ・ライブラリのインストール

#### Python 3.11以上

```bash
brew install python
```

インストール確認：`python3 --version`（`Python 3.11` 以上が表示されればOK）

#### Google Chrome

https://www.google.com/chrome/ からダウンロードしてインストールしてください。

> このツールは Chrome をリモート操作してスクレイピングを行うため、Chrome が必須です。

#### Pythonライブラリ（requirements.txt）

ターミナルで `aucfan_tool` フォルダに移動してから実行：

```bash
cd ~/Downloads/aucfan_tool
pip install -r requirements.txt --break-system-packages
```

`requirements.txt` に含まれる主なライブラリ：

| ライブラリ | 役割 |
|---|---|
| `flask` | Webサーバー（管理画面の表示） |
| `selenium` | ChromeをPythonから自動操作 |
| `imagehash` | 画像のpHash計算（同一商品グループ化） |
| `Pillow` | 画像処理 |
| `google-generativeai` | Gemini AIによる除外判定 |
| `pandas` | CSVの読み書き・データ操作 |
| `python-dotenv` | `.env` ファイルの読み込み |
| `requests` | 画像のダウンロード |
| `beautifulsoup4` | HTMLの解析 |

### 1-3. Gemini APIキーの取得と設定

#### APIキーの取得手順

1. ブラウザで https://aistudio.google.com/ を開く
2. Googleアカウントでサインイン
3. 左メニューの「**Get API key**」をクリック
4. 「**Create API key**」ボタンを押す
5. 表示されたキー（`AIzaSy...` で始まる文字列）をコピーする

#### `.env` ファイルへの保存

`aucfan_tool` フォルダの中に `.env` というファイルがあります（隠しファイルのため Finder では非表示）。テキストエディタで開いて以下を記載：

```
GEMINI_API_KEY=AIzaSy...（取得したキーを貼り付け）
```

ターミナルから作成・編集する場合：

```bash
echo 'GEMINI_API_KEY=AIzaSy...' >> ~/Downloads/aucfan_tool/.env
```

> **セキュリティについて**: `.env` ファイルは `.gitignore` に登録されているため、`git push` しても GitHub にはアップロードされません。APIキーが外部に漏れる心配はありません。

## 2. ファイル構成

> **コードを触らない方もここだけ把握しておくと便利です。** どのファイルが何の役割を持っているか知っておくと、設定変更や困ったときのトラブルシュートがスムーズになります。

### プログラムファイル（Pythonコード）

| ファイル名 | 役割 |
|---|---|
| `app.py` | アプリのサーバー本体。起動・ボタン操作の受け取り・各処理の指示出しを担う中心ファイル |
| `scraper.py` | AucFanをスクレイピングするメインプログラム。Chromeを自動操作して商品一覧・詳細を取得する |
| `seller_analyzer.py` | STEP 2専用。セラーIDごとに商品ページを巡回して取得する処理 |
| `sellers_master.py` | マスターセラーリスト（`data/sellers_master.json`）の読み書きを管理する |
| `data_manager.py` | セッションフォルダの作成・商品データの保存・CSV書き出しを管理する |
| `gemini_client.py` | Gemini AIに「この商品は除外すべきか？」を問い合わせる処理 |
| `image_processor.py` | 商品画像をダウンロードしてpHash（画像の類似度判定）を計算し、同一商品をグループ化する |
| `config.py` | 価格帯・待機時間・除外キーワードリストなど**全設定値が集まっているファイル**。設定を変えたいときはここか`.env`を見る |

### 設定・環境ファイル

| ファイル名 | 役割 |
|---|---|
| `.env` | APIキー・価格帯・件数設定など個人ごとの設定値。**GitHubにはアップロードされない**（`.gitignore`で除外済み） |
| `start.sh` | アプリの起動スクリプト。ChromeをデバッグモードでいちどRelaunchしてFlaskを立ち上げる |
| `requirements.txt` | このアプリに必要なPythonライブラリ一覧。`pip install -r requirements.txt` で一括インストールできる |

### 画面ファイル（フロントエンド）

| ファイル名 | 役割 |
|---|---|
| `templates/index.html` | ブラウザに表示されるアプリの画面HTML |
| `static/app.js` | 画面のボタン動作・グリッド表示・タブ切り替えなどの処理 |
| `static/style.css` | 画面のデザイン（色・レイアウト・フォント等） |

### データフォルダ

| フォルダ / ファイル | 役割 |
|---|---|
| `リサーチ結果/` （ローカル or Google Drive） | スクレイピング結果の保存先。デフォルトは Google Drive の `AucFanToolData/リサーチ結果/`。`S1_YYYYMMDD_XX_キーワード/`・`S2_/`・`S3_/` という名前のフォルダが増えていく |
| `Google Drive/AucFanToolData/sellers_master.json` | マスターセラーリストのデータ本体。STEP 1完了のたびに自動更新される（Google Drive 未接続時は `data/sellers_master.json` にフォールバック） |
| `docs/` | 本書（`USER_GUIDE.md`）とエンジニア向けの`CODE_GUIDE.md`が入っているフォルダ |

> **よく変更するファイル TOP 3**: ①`.env`（価格・件数の調整）、②`config.py`（除外キーワードの追加・CSSセレクターの修正）、③`prompts.yaml`（Geminiへの指示文の調整）

## 3. アプリの概要

**AucFan リサーチツール**は、ヤフオク・メルカリ等の落札データを集計しているサイト「AucFan」から商品情報を自動収集し、**仕入れ候補となるノーブランド商品**を効率よく発見するためのツールです。

### 主な機能

| 機能 | 内容 |
|------|------|
| STEP 1 キーワードリサーチ | キーワードで検索 → 同一商品をグループ化 → 仕入れ候補を自動抽出 |
| STEP 2 セラーリサーチ | 気になるセラーの全商品を横断スクレイピング |
| STEP 3 マスターリスト横断 | 蓄積したセラーIDをまとめてリサーチ |
| AI除外判定 | 食品・ブランド品・危険物等を Gemini AI が自動除外 |
| 結果エクスポート | CSV・HTML形式でデータを書き出し |

### 仕組みのイメージ

```
AucFan（ブラウザで開いた状態）
        ↓  Selenium で自動操作
  商品データ収集（1ページ50件×数百ページ）
        ↓
  同一商品グループ化（pHash画像認識）
        ↓
  AI除外判定（Gemini Vision/テキスト）
        ↓
  結果グリッドに表示 → セラーIDがマスターリストに自動追加
```

## 4. 起動方法

### 手順

1. **Macのターミナルを開く**（Finder → アプリケーション → ユーティリティ → ターミナル）

2. **ツールフォルダに移動してスクリプトを実行する**

   ```bash
   bash ~/Downloads/aucfan_tool/start.sh
   ```

3. **自動で以下が実行される**
   - Chromeが一度終了し、デバッグモードで再起動
   - Flaskサーバーが起動（ポート5001）
   - ブラウザで `http://localhost:5001` が自動的に開く

4. **Chromeの新しいタブで AucFan を開く**
   - `https://aucfan.com` にアクセスしてログイン
   - 検索したいキーワードで検索し、**1ページ目を表示した状態にする**

5. **管理画面のタブに戻ってスクレイピングを操作する**

> **iPhone/iPad からも操作可能** — 同じWi-Fi上であれば `http://[MacのIPアドレス]:5001` でアクセスできます。

### 停止方法

ターミナルで `Ctrl + C` を押すと Flask サーバーが停止します（Chromeはそのまま残ります）。

## 5. STEP 1 — キーワードリサーチ

AucFanで特定のキーワードを検索し、同一商品が複数出品されているグループを探します。

### 手順

#### Macの場合（推奨）

1. 「STEP 1 キーワードリサーチ」タブを選択
2. 「キーワード」欄に検索語を入力（例：`LEDライト`）
3. Chromeで AucFan を開き、同じキーワードで検索して1ページ目を表示しておく
4. **「▶ スクレイピング開始」**をクリック
5. 進捗バーが表示され、商品が順次グリッドに追加される
6. 「■ 停止」で任意のタイミングで中断可能

#### iPhone/iPad からの場合

- **方法①（推奨）**：SafariでAucFanを検索 → URLをコピー → 「AucFan 検索URLを貼り付ける」欄に貼り付け → スクレイピング開始
- **方法②**：「🔍 Chromeのタブを確認」ボタンを押してMacで開いているAucFanタブを選択

### 進捗・完了表示

スクレイピング中は以下の表示でリアルタイムに確認できます。

- 画面上に「**X件 / Y件処理済み**」というカウンターが表示されます
- 完了すると画面上部に緑色の「**✅ スクレイピング完了（N件処理）**」バナーが表示されます（✕ボタンで閉じられます）
- ターミナルにも「`=== STEP 1 スクレイピング完了 === 全N件処理`」という目立つログが出力されます

### スクレイピング完了後

- 結果グリッドに商品グループが表示される
- **セラーIDがマスターリストに自動追加**（同一商品グループが2件以上のセラーのみ）
- 結果はセッションフォルダ（`リサーチ結果/S1_YYYYMMDD_01_キーワード/`）に自動保存される

### 結果の確認

| 表示 | 意味 |
|------|------|
| 🔵 仕入れ候補 | 同一商品が5件以上のグループ（設定変更可） |
| 🟡 次期候補 | 同一商品が4件以上のグループ |
| ⏳ 確認待ち | AI判定前の商品 |
| ⚠️ 要確認 | AIが安全リスクフラグを立てた商品 |
| ✅ OK | ユーザーが承認した商品 |
| ❌ NG | 除外した商品 |

## 6. STEP 2 — セラーリサーチ

特定のセラーの全商品をスクレイピングし、セラーをまたいで同一商品を比較します。

> **新品のみ取得**：STEP 2・STEP 3では AucFan 一覧ページの「商品状態」欄が「新品・未使用・未開封・未着用」の商品のみを取得します（Amazon中国輸入リサーチ用途のため、中古品は自動除外）。中古品を含めたい場合は `.env` に `SELLER_NEW_ONLY=false` を追加してください。

### 手順

1. 「STEP 2 セラーリサーチ」タブを選択

2. **セラーIDの取得方法を選ぶ（3通りあります）**

   **⚡ 最速（推奨）**：「🔗 現在のSTEP 1結果からセラーIDを取得」ボタンを押す
   - STEP 1の結果がメモリにある場合、CSVなしで直接取得できる

   **Google DriveのSTEP 1セッションから**：🔄ボタンでセッション一覧を更新 → 使いたいセッションを選択

   **CSVファイルから**：「💾 最新の保存済みCSVを使用」または「📂 CSVファイルを選択」

3. セラーIDの一覧が表示されたら「**▶ スクレイピング開始**」をクリック

4. 各セラーの商品が順次スクレイピングされ、全完了後に結果グリッドに表示される

### 進捗・完了表示

- 画面上に「**X件 / Y件処理済み**」というカウンターが表示されます
- 完了すると画面上部に緑色の「**✅ スクレイピング完了（N件処理）**」バナーが表示されます（✕ボタンで閉じられます）
- ターミナルにも「`=== STEP 2 スクレイピング完了 === 全N件処理`」という目立つログが出力されます
- 中古品を除外したページは「`[商品状態フィルタ] N件除外（中古等）→ 次ページへ`」とログに表示され、自動で次ページに進みます
- **ログイン切れを検知した場合**はヘッダーが「⚠️ ログイン待ち」に変わり、バナーに「🔄 今すぐ確認して再開」ボタンが表示されます。ログイン後は自動（30秒）またはボタン押下で即再開します

### 中古セラーの自動スキップ

1セラーのスクレイピング中に中古商品がデフォルト **3件** を超えると、そのセラーを「中古主体セラー」と判断して打ち切り、取得済みデータを自動削除して次のセラーへ進みます。

- セラーリストに **「🚫 中古N件超でスキップ」**（オレンジ・件数入り）と表示されます
- ターミナルにも理由と件数が表示されます（例：`🚫 中古セラースキップ: seller_abc123 / 中古件数: 7件（閾値: 5件超でスキップ）`）
- 閾値は `.env` の `SELLER_USED_SKIP_THRESHOLD` で変更できます（デフォルト：`5`）
- `0` に設定するとスキップ無効（アイテム単位フィルターのみ）

> **なぜ5件許容するか**：中国輸入が主力のセラーでも、過去に1〜2件だけ中古を出品したことがある場合があります。1件でも即スキップすると正規セラーを誤除外するおそれがあるため、デフォルト5件（6件目でスキップ）の許容幅を設けています。

### 結果の確認

セラー分析モードでは、同一商品をセラーをまたいでグループ化して表示します。セラー分析専用フィルターで件数を絞り込めます。

> **注意**：STEP 1でキーワード＋新品検索をしてもセラーが新品・中古両方を出品している場合があります。STEP 2ではそのセラーの全出品ページを巡回しますが、商品状態フィルタ・中古セラースキップにより中古品は自動除外されます。

## 7. STEP 3 — マスターリスト横断リサーチ

マスターリストに蓄積された「未スクレイピング」のセラーIDをまとめてリサーチします。STEP 1を繰り返すたびにセラーが自動追加されていくため、定期的にSTEP 3を実行することで効率よく新規セラーを発掘できます。

### 手順

1. 「STEP 3 マスターセラーリサーチ」タブを選択

2. **設定を確認する**

   | 設定 | 説明 |
   |------|------|
   | 並び順 | 「新しい順」= 最近追加されたセラーから処理／「古い順」= 古いセラーから処理 |
   | バッチサイズ | 1回の実行で処理するセラー数（50/100/200/上限なし） |

3. 「対象: ○○件」のプレビューを確認して「**▶ スクレイピング開始**」をクリック

4. スクレイピングが完了すると結果グリッドに表示される

### 進捗・完了表示

- 画面上に「**X件 / Y件処理済み**」というカウンターが表示されます
- 完了すると画面上部に緑色の「**✅ スクレイピング完了（N件処理）**」バナーが表示されます（✕ボタンで閉じられます）
- ターミナルにも「`=== STEP 3 スクレイピング完了 === 全N件処理`」という目立つログが出力されます
- **ログイン切れを検知した場合**はSTEP 2と同様にバナーと「🔄 今すぐ確認して再開」ボタンが表示されます

> 未スクレイピングのセラーが0件の場合は実行できません。STEP 1でキーワードリサーチを行うとセラーが追加されます。

## 10. 除外ルールについて

スクレイピングした商品は複数の段階で除外判定が行われます。

### 除外される商品の種類

**タイトルキーワード除外**（スクレイピング時点で弾く）
- チケット・金券・優待券
- 食品・食材・飲料全般（水産物・農産物・肉類・お菓子・酒類 等）
- 危険物・高圧ガス（エアコンガス・スプレー缶 等）
- 特定球団グッズ・サイン色紙 等
- **身体への影響リスク商品**：シャンプー・リンス・トリートメント・育毛剤・ヘアカラー・マスク・フェイスパック・化粧品・スキンケア・ボディソープ・除毛クリーム・制汗剤・歯磨き粉・サプリメント・医薬品・消毒液 等
- **生体・動物**：ニワトリ品種（「有髭」等のbearded品種含む）・観賞魚・メダカ・カブトムシ・インコ・ハムスター 等（Amazon輸入対象外のため除外）
- **植物の苗・種子**：多肉植物・サボテン・球根 等

**タイトル先頭メーカー名除外**
- タイトルの先頭1〜2番目にHITACHI・Panasonic・Sony・Apple等の有名メーカー名が来る商品
- ただし、**自動車・バイク関連キーワードを含む商品はこの除外対象外**（下記「自動車・バイク関連のバイパスルール」を参照）

**自動車・バイク関連のバイパスルール**
- 自動車・バイク・カー用品に関連するキーワード（車種名・「カー用品」「バイク用品」等）が商品タイトルに含まれる場合、メーカー名が先頭にあっても除外されません
- **理由**: カー用品・バイク用品はメーカー互換品・汎用品が多く、ブランド名が商品説明に含まれていても仕入れ対象になりうるためです
- **例**:
  - 「トヨタ キーケース」→ 除外されない（カー用品系キーワードにより除外をバイパス）
  - 「トヨタ 純正部品」→ 除外される（「純正部品」は自動車バイパスキーワードに含まれないため通常の除外対象）

**Gemini AI テキスト判定**（`詳細取得あり`モードのみ）
- 絶対除外カテゴリ（食品・衣類・AC100V家電・危険物・トレカ本体）
- 要確認フラグ（車の保安部品等）
- 有名ブランド品

**Gemini AI Vision判定**（同一商品グループが4件以上のグループ）
- 画像を直接見て総合判定（1回のAPIコールで除外・要確認・ブランド品を判定）

> Gemini APIキーが設定されていない場合はAI判定がスキップされ、pHash（画像ハッシュ）によるグループ化のみで動作します。

## 12. よくある質問

### AucFanのログインが切れてしまった（複数Macで同時ログインした場合など）

AucFanは2台のMacで同じアカウントにログインすると、先にログインしていた側がゲスト状態になることがあります。スクレイピング中にログアウトが検知されると、自動的に以下の動作をします。

**自動検知と待機：**
1. ページ上に「ようこそ、 ゲストさん」の文言を検出するとログアウトと判断
2. ヘッダーが「**⚠️ ログイン待ち**」に切り替わる
3. 画面上部に**琥珀色のバナー**「AucFanのログインが切れました」が表示される
4. ターミナルにも警告と再開予定URLが表示される
5. **30秒ごとに自動でログイン状態を確認**し、ログインを検知したら自動で再開する

**すぐに再開したい場合：**
- バナー右端の「**🔄 今すぐ確認して再開**」ボタンを押す（タブ移動なし・即時確認）

**注意点：**
- タブはSTEP 2のままで移動しません（LocalStorageで位置を保持）
- スクレイピング再開後、pHash画像グループ化・Vision判定も自動で続きます
- STEP 1・STEP 2・STEP 3すべてで動作します

---

### スクレイピングが突然止まった

**原因として多いもの：**
- AucFanがメンテナンス中またはサーバーエラー
- Chromeのセッションが切れた
- ネットワークが不安定

**対処法：**
1. 「■ 停止」ボタンを押して完全に停止させる
2. AucFanのページをリロードする
3. 再度「▶ スクレイピング開始」を押す（途中から再開される場合があります）

---

### 商品が少ししか表示されない

**考えられる原因：**
- 検索結果自体が少ない（AucFanで検索してページ数を確認）
- 価格フィルターで多くの商品が除外されている（デフォルト: 1,000〜3,000円）
- 同一商品グループが少ない（グループ化の閾値が高い）

**対処法：**
- `.env` ファイルの `MIN_PRICE` / `MAX_PRICE` を変更して価格範囲を広げる
- フィルターバーの「最小グループ数」を「1」に下げて全商品を表示する

---

### マスターリストへの追加条件は？

STEP 1完了時に、pHashグループ化後の**グループサイズが2件以上**のグループに含まれるセラーIDのみが追加されます。1件のみの商品（ユニーク品）のセラーは追加されません。

この閾値は `.env` の `MASTER_SELLER_MIN_GROUP_SIZE` で変更できます（デフォルト: 2）。

---

### AI判定が動かない

Gemini APIキーが設定されていない、または無効の可能性があります。

**確認方法：**
1. `~/Downloads/aucfan_tool/.env` ファイルをテキストエディタで開く
2. `GEMINI_API_KEY=` の後ろにAPIキーが入っているか確認
3. キーがない場合は [Google AI Studio](https://aistudio.google.com/app/apikey) で取得して設定

AI判定なしでも pHash による画像グループ化は動作します。

---

### 画面上部にエラーのバナーが表示された

スクレイピング中にGemini APIでエラーが発生すると、画面上部にオレンジ・赤・黄色のバナーが表示されます。✕ボタンで閉じることができます。

以下はエラーの種類と対処法の一覧です。

| エラー種別 | 表示色 | 意味 | 対処法 |
|---|---|---|---|
| レート制限超過（429） | オレンジ〜赤 | 無料枠の1日または1分あたりのリクエスト上限に達した | しばらく待つか、Google AI Studioで従量課金に切り替える |
| 一時混雑（503） | 黄色 | Geminiサーバーが混雑している | 数分待って再スクレイピングする |
| 内部エラー（500） | 黄色 | Gemini側で一時的なエラーが発生した | 判定はスキップして続行している。再スクレイピングで回復することが多い |
| APIキーエラー（403） | 赤 | APIキーの権限不足または無効 | `.env` の `GEMINI_API_KEY` を確認する。必要に応じて [Google AI Studio](https://aistudio.google.com/app/apikey) で新しいキーを発行する |
| 入力エラー（400） | 黄色 | 画像が壊れているなど入力データの問題 | 該当商品はスキップして続行している。特別な対応は不要 |

> **スクレイピングは止まらない**: 上記エラーのうち503・500・400は、AI判定をスキップしてスクレイピング自体は継続されます。429（レート制限）と403（権限エラー）のみ、AI判定が一時停止または不能になる場合があります。

**無料枠について**

Gemini APIの無料枠は1日1,500リクエスト・1分あたり15リクエストが目安です。毎日複数回スクレイピングを行う場合は従量課金（月数百円程度）への切り替えを検討してください。429エラーが頻発するようになったら切り替えのサインです。従量課金への切り替えは [Google AI Studio](https://aistudio.google.com/app/apikey) の課金設定から行えます。

### 結果をiPhoneで確認したい

3つの方法があります。

**方法①：同じWi-Fiで管理画面を直接開く（最も確実・推奨）**
1. MacとiPhoneを同じWi-Fiに接続
2. MacのIPアドレスを確認（ターミナルで `ipconfig getifaddr en0`）
3. iPhoneのSafariで `http://[MacのIPアドレス]:5001` を開く

> Macアプリが起動していればリアルタイムでアプリと同じ画面が見られます。ファイル転送不要で最も軽快です。

**方法②：Google DriveのHTMLファイルをSafariで開く（オフライン確認用）**
1. フィルターバーの「📱 HTML保存（iPhone用）」または「📱 HTML保存（候補のみ・軽量）」をクリック
2. Mac上の Google Drive `AucFanToolData` フォルダに自動保存されるので、iPhoneの Google Drive アプリで同期を確認
3. iPhoneのファイルアプリまたはGoogle Driveアプリでファイルをタップ → **共有ボタン（□↑）→「Safariで開く」** を選択

> 件数が多くてSafariで開けない場合は「候補のみ・軽量」ボタンを使うとファイルサイズが大幅に小さくなります。画像は120×120px・JPEG圧縮済みのため、元データより数分の1のサイズになります。

**方法③：AirDropで直接送る**
1. 上記②と同様にHTMLを生成
2. MacのFinderで `AucFanToolData` フォルダ内のHTMLファイルを右クリック →「共有」→「AirDrop」でiPhoneに送信
3. iPhoneで受信後、ファイルアプリを開き 共有ボタン →「Safariで開く」

> いずれの方法でも、HTMLファイルはデータの読み込みには使えません（閲覧専用）。データを別のMacや別のアプリで再利用したい場合はCSVをお使いください。

### CSV読み込み時にHTMLファイルを誤って選択してしまった

CSV読み込みボタンでHTMLファイルを選択した場合、赤いトーストメッセージが表示されてアップロードは自動的にキャンセルされます。

## 13. Google Drive連携（複数Mac共有）

セッションデータ（スクレイピング結果・画像）および**マスターセラーリスト**の保存先は**デフォルトで Google Drive** に設定されています。Google Drive for Desktop をインストールしてサインインするだけで、`.env` の変更なしに2拠点からデータを共有できます。

> Google Drive が未接続の場合は自動的にローカル（`リサーチ結果/` / `data/sellers_master.json`）にフォールバックします。

### 13-1. Google Drive for Desktop のインストール

1. https://www.google.com/drive/download/ から「Google Drive for Desktop」をダウンロード・インストールする
2. インストール後、`shinozakistore@gmail.com` でサインインする
3. 「マイドライブをこのMacにミラーリング」を選択して同期を開始する

マウント確認（以下を実行してフォルダが表示されれば OK）：

```bash
ls ~/Library/CloudStorage/
```

`GoogleDrive-shinozakistore@gmail.com` が表示されれば正常です。

### 13-2. データ保存先

| 保存先 | 内容 |
|---|---|
| `Google Drive/マイドライブ/AucFanToolData/リサーチ結果/` | セッションフォルダ（items.json・results.csv・images/） |
| `Google Drive/マイドライブ/AucFanToolData/` | 閲覧用HTML（_Mac表示用・_iPhone表示用） |
| `Google Drive/マイドライブ/AucFanToolData/sellers_master.json` | マスターセラーリスト（2拠点で共有） |

> フォルダは初回スクレイピング時に自動作成されます。事前作成は不要です。

### 13-3. 動作確認

アプリを起動して STEP 1 を実行し、Google Drive の `AucFanToolData/リサーチ結果/` にセッションフォルダが作成されれば設定完了です。アプリの「更新」ボタンを押すとセッション一覧に表示されます。

### 13-4. 2台目 Mac への適用

2台目でも [§1-6](#1-6-別端末2台目macへのセットアップ) の手順でセットアップし、同じ Google アカウントで Google Drive for Desktop にサインインするだけで同じデータを参照できます。`.env` の変更は不要です。

> **注意**: 2台のMacで同時にスクレイピングを実行しないでください。同一ファイルへの同時書き込みでデータが破損するおそれがあります。

### 13-5. ローカル保存に切り替えたい場合

Google Drive を使わずローカルに保存したい場合は `.env` に以下を追加します：

```
OUTPUT_BASE_DIR=リサーチ結果
```
