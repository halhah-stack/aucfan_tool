## 1. 導入準備

> はじめてこのツールを使う方向けのセットアップ手順です。セットアップ済みの方は [2. アプリの概要](#2-アプリの概要) へ進んでください。

### 1-1. Homebrew のインストール

**Homebrew** は macOS 用のパッケージ管理ツールです。Python や Git などのソフトウェアをターミナルから簡単にインストールできます。

**インストール**（公式サイトのコマンドを1行コピーして実行）：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

> ターミナルを開くには：Finder → アプリケーション → ユーティリティ → ターミナル

**インストール確認**：

```bash
brew --version
```

`Homebrew 4.x.x` のように表示されれば成功です。

---

### 1-2. 必要なアプリ・ライブラリのインストール

#### Python 3.11以上

```bash
brew install python
```

インストール確認：`python3 --version`（`Python 3.11` 以上が表示されればOK）

#### Google Chrome

https://www.google.com/chrome/ からダウンロードしてインストールしてください。

> このツールは Chrome をリモート操作してスクレイピングを行うため、Chrome が必須です。

#### Pythonライブラリ（requirements.txt）

ターミナルで `aucfan_tool` フォルダに移動してから実行：

```bash
cd ~/Downloads/aucfan_tool
pip install -r requirements.txt --break-system-packages
```

`requirements.txt` に含まれる主なライブラリ：

| ライブラリ | 役割 |
|---|---|
| `flask` | Webサーバー（管理画面の表示） |
| `selenium` | ChromeをPythonから自動操作 |
| `imagehash` | 画像のpHash計算（同一商品グループ化） |
| `Pillow` | 画像処理 |
| `google-generativeai` | Gemini AIによる除外判定 |
| `pandas` | CSVの読み書き・データ操作 |
| `python-dotenv` | `.env` ファイルの読み込み |
| `requests` | 画像のダウンロード |
| `beautifulsoup4` | HTMLの解析 |

### 1-3. Gemini APIキーの取得と設定

#### APIキーの取得手順

1. ブラウザで https://aistudio.google.com/ を開く
2. Googleアカウントでサインイン
3. 左メニューの「**Get API key**」をクリック
4. 「**Create API key**」ボタンを押す
5. 表示されたキー（`AIzaSy...` で始まる文字列）をコピーする

#### `.env` ファイルへの保存

`aucfan_tool` フォルダの中に `.env` というファイルがあります（隠しファイルのため Finder では非表示）。テキストエディタで開いて以下を記載：

```
GEMINI_API_KEY=AIzaSy...（取得したキーを貼り付け）
```

ターミナルから作成・編集する場合：

```bash
echo 'GEMINI_API_KEY=AIzaSy...' >> ~/Downloads/aucfan_tool/.env
```

> **セキュリティについて**: `.env` ファイルは `.gitignore` に登録されているため、`git push` しても GitHub にはアップロードされません。APIキーが外部に漏れる心配はありません。

## 2. ファイル構成

> **コードを触らない方もここだけ把握しておくと便利です。** どのファイルが何の役割を持っているか知っておくと、設定変更や困ったときのトラブルシュートがスムーズになります。

### プログラムファイル（Pythonコード）

| ファイル名 | 役割 |
|---|---|
| `app.py` | アプリのサーバー本体。起動・ボタン操作の受け取り・各処理の指示出しを担う中心ファイル |
| `scraper.py` | AucFanをスクレイピングするメインプログラム。Chromeを自動操作して商品一覧・詳細を取得する |
| `seller_analyzer.py` | STEP 2専用。セラーIDごとに商品ページを巡回して取得する処理 |
| `sellers_master.py` | マスターセラーリスト（`data/sellers_master.json`）の読み書きを管理する |
| `data_manager.py` | セッションフォルダの作成・商品データの保存・CSV書き出しを管理する |
| `gemini_client.py` | Gemini AIに「この商品は除外すべきか？」を問い合わせる処理 |
| `image_processor.py` | 商品画像をダウンロードしてpHash（画像の類似度判定）を計算し、同一商品をグループ化する |
| `config.py` | 価格帯・待機時間・除外キーワードリストなど**全設定値が集まっているファイル**。設定を変えたいときはここか`.env`を見る |

### 設定・環境ファイル

| ファイル名 | 役割 |
|---|---|
| `.env` | APIキー・価格帯・件数設定など個人ごとの設定値。**GitHubにはアップロードされない**（`.gitignore`で除外済み） |
| `start.sh` | アプリの起動スクリプト。ChromeをデバッグモードでいちどRelaunchしてFlaskを立ち上げる |
| `requirements.txt` | このアプリに必要なPythonライブラリ一覧。`pip install -r requirements.txt` で一括インストールできる |

### 画面ファイル（フロントエンド）

| ファイル名 | 役割 |
|---|---|
| `templates/index.html` | ブラウザに表示されるアプリの画面HTML |
| `static/app.js` | 画面のボタン動作・グリッド表示・タブ切り替えなどの処理 |
| `static/style.css` | 画面のデザイン（色・レイアウト・フォント等） |

### データフォルダ

| フォルダ / ファイル | 役割 |
|---|---|
| `リサーチ結果/` | スクレイピング結果の保存先。`S1_YYYYMMDD_XX_キーワード/`・`S2_/`・`S3_/` という名前のフォルダが増えていく |
| `data/sellers_master.json` | マスターセラーリストのデータ本体。STEP 1完了のたびに自動更新される |
| `docs/` | 本書（`USER_GUIDE.md`）とエンジニア向けの`CODE_GUIDE.md`が入っているフォルダ |

> **よく変更するファイル TOP 3**: ①`.env`（価格・件数の調整）、②`config.py`（除外キーワードの追加・CSSセレクターの修正）、③`prompts.yaml`（Geminiへの指示文の調整）

## 3. アプリの概要

**AucFan リサーチツール**は、ヤフオク・メルカリ等の落札データを集計しているサイト「AucFan」から商品情報を自動収集し、**仕入れ候補となるノーブランド商品**を効率よく発見するためのツールです。

### 主な機能

| 機能 | 内容 |
|------|------|
| STEP 1 キーワードリサーチ | キーワードで検索 → 同一商品をグループ化 → 仕入れ候補を自動抽出 |
| STEP 2 セラーリサーチ | 気になるセラーの全商品を横断スクレイピング |
| STEP 3 マスターリスト横断 | 蓄積したセラーIDをまとめてリサーチ |
| AI除外判定 | 食品・ブランド品・危険物等を Gemini AI が自動除外 |
| 結果エクスポート | CSV・HTML形式でデータを書き出し |

### 仕組みのイメージ

```
AucFan（ブラウザで開いた状態）
        ↓  Selenium で自動操作
  商品データ収集（1ページ50件×数百ページ）
        ↓
  同一商品グループ化（pHash画像認識）
        ↓
  AI除外判定（Gemini Vision/テキスト）
        ↓
  結果グリッドに表示 → セラーIDがマスターリストに自動追加
```

## 4. 起動方法

### 手順

1. **Macのターミナルを開く**（Finder → アプリケーション → ユーティリティ → ターミナル）

2. **ツールフォルダに移動してスクリプトを実行する**

   ```bash
   bash ~/Downloads/aucfan_tool/start.sh
   ```

3. **自動で以下が実行される**
   - Chromeが一度終了し、デバッグモードで再起動
   - Flaskサーバーが起動（ポート5001）
   - ブラウザで `http://localhost:5001` が自動的に開く

4. **Chromeの新しいタブで AucFan を開く**
   - `https://aucfan.com` にアクセスしてログイン
   - 検索したいキーワードで検索し、**1ページ目を表示した状態にする**

5. **管理画面のタブに戻ってスクレイピングを操作する**

> **iPhone/iPad からも操作可能** — 同じWi-Fi上であれば `http://[MacのIPアドレス]:5001` でアクセスできます。

### 停止方法

ターミナルで `Ctrl + C` を押すと Flask サーバーが停止します（Chromeはそのまま残ります）。

## 5. STEP 1 — キーワードリサーチ

AucFanで特定のキーワードを検索し、同一商品が複数出品されているグループを探します。

### 手順

#### Macの場合（推奨）

1. 「STEP 1 キーワードリサーチ」タブを選択
2. 「キーワード」欄に検索語を入力（例：`LEDライト`）
3. Chromeで AucFan を開き、同じキーワードで検索して1ページ目を表示しておく
4. **「▶ スクレイピング開始」**をクリック
5. 進捗バーが表示され、商品が順次グリッドに追加される
6. 「■ 停止」で任意のタイミングで中断可能

#### iPhone/iPad からの場合

- **方法①（推奨）**：SafariでAucFanを検索 → URLをコピー → 「AucFan 検索URLを貼り付ける」欄に貼り付け → スクレイピング開始
- **方法②**：「🔍 Chromeのタブを確認」ボタンを押してMacで開いているAucFanタブを選択

### スクレイピング完了後

- 結果グリッドに商品グループが表示される
- **セラーIDがマスターリストに自動追加**（同一商品グループが2件以上のセラーのみ）
- 結果はセッションフォルダ（`リサーチ結果/S1_YYYYMMDD_01_キーワード/`）に自動保存される

### 結果の確認

| 表示 | 意味 |
|------|------|
| 🔵 仕入れ候補 | 同一商品が5件以上のグループ（設定変更可） |
| 🟡 次期候補 | 同一商品が4件以上のグループ |
| ⏳ 確認待ち | AI判定前の商品 |
| ⚠️ 要確認 | AIが安全リスクフラグを立てた商品 |
| ✅ OK | ユーザーが承認した商品 |
| ❌ NG | 除外した商品 |

## 6. STEP 2 — セラーリサーチ

特定のセラーの全商品をスクレイピングし、セラーをまたいで同一商品を比較します。

### 手順

1. 「STEP 2 セラーリサーチ」タブを選択

2. **セラーIDの取得方法を選ぶ（3通りあります）**

   **⚡ 最速（推奨）**：「🔗 現在のSTEP 1結果からセラーIDを取得」ボタンを押す
   - STEP 1の結果がメモリにある場合、CSVなしで直接取得できる

   **過去のSTEP 1セッションから**：🔄ボタンでセッション一覧を更新 → 使いたいセッションを選択

   **CSVファイルから**：「💾 最新の保存済みCSVを使用」または「📂 CSVファイルを選択」

3. セラーIDの一覧が表示されたら「**▶ スクレイピング開始**」をクリック

4. 各セラーの商品が順次スクレイピングされ、全完了後に結果グリッドに表示される

### 結果の確認

セラー分析モードでは、同一商品をセラーをまたいでグループ化して表示します。セラー分析専用フィルターで件数を絞り込めます。

## 7. STEP 3 — マスターリスト横断リサーチ

マスターリストに蓄積された「未スクレイピング」のセラーIDをまとめてリサーチします。STEP 1を繰り返すたびにセラーが自動追加されていくため、定期的にSTEP 3を実行することで効率よく新規セラーを発掘できます。

### 手順

1. 「STEP 3 マスターセラーリサーチ」タブを選択

2. **設定を確認する**

   | 設定 | 説明 |
   |------|------|
   | 並び順 | 「新しい順」= 最近追加されたセラーから処理／「古い順」= 古いセラーから処理 |
   | バッチサイズ | 1回の実行で処理するセラー数（50/100/200/上限なし） |

3. 「対象: ○○件」のプレビューを確認して「**▶ スクレイピング開始**」をクリック

4. スクレイピングが完了すると結果グリッドに表示される

> 未スクレイピングのセラーが0件の場合は実行できません。STEP 1でキーワードリサーチを行うとセラーが追加されます。

## 8. マスターリストタブ

STEP 1実行のたびに自動追加されるセラーIDを管理するタブです。

### 表示項目

| 列 | 説明 |
|----|------|
| セラーID | AucFanのセラー識別子 |
| 初回検出日 | このセラーが最初にマスターリストに追加された日付 |
| 最終スクレイピング | 最後にSTEP 3でスクレイピングした日付（未実施は「未」） |
| 候補件数 | 最後のSTEP 3で見つかった仕入れ候補の件数 |
| キーワード | 初回検出時のSTEP 1キーワード |

### 操作ボタン

| ボタン | 機能 |
|--------|------|
| 🔄 更新 | リストを最新状態に更新 |
| 💾 CSV保存 | マスターリスト全件をCSVでダウンロード |
| 📄 HTML書き出し | 検索機能付きスタンドアロンHTMLを書き出し（iPhone等で見る用） |
| 📂 CSV読み込み | 外部のCSVファイルからセラーIDをインポート（`seller_id`列が必要） |
| 🗑 全削除 | マスターリストを全件削除（確認あり） |
| 各行の 🗑 | その行のセラーIDを個別削除（ホバーで表示） |

### ページ送り

グリッドの右上・右下にページネーションが表示されます。1ページあたり30グループが表示されます。

### エクスポート

|--------|------|

| 📂 CSV読み込み | 保存済みのCSVをグリッドに再表示 |

## 10. 除外ルールについて

スクレイピングした商品は複数の段階で除外判定が行われます。

### 除外される商品の種類

**タイトルキーワード除外**（スクレイピング時点で弾く）
- チケット・金券・優待券
- 食品・食材・飲料全般（水産物・農産物・肉類・お菓子・酒類 等）
- 危険物・高圧ガス（エアコンガス・スプレー缶 等）
- 特定球団グッズ・サイン色紙 等

**タイトル先頭メーカー名除外**
- タイトルの先頭1〜2番目にHITACHI・Panasonic・Sony・Apple等の有名メーカー名が来る商品

**Gemini AI テキスト判定**（`詳細取得あり`モードのみ）
- 絶対除外カテゴリ（食品・衣類・AC100V家電・危険物・トレカ本体）
- 要確認フラグ（車の保安部品等）
- 有名ブランド品

**Gemini AI Vision判定**（同一商品グループが4件以上のグループ）
- 画像を直接見て総合判定（1回のAPIコールで除外・要確認・ブランド品を判定）

> Gemini APIキーが設定されていない場合はAI判定がスキップされ、pHash（画像ハッシュ）によるグループ化のみで動作します。

## 11. 過去セッションの表示方法

各STEPパネルの下部に「📂 過去のセッション」セクションがあります。

### 操作手順

1. 目的のSTEPタブを開く（例：STEP 1）
2. 「📂 過去のSTEP 1セッション」セクションの「🔄 更新」ボタンを押す
3. セッション一覧が表示される
4. 表示したいセッションの「**📂 表示**」ボタンを押す
5. そのセッションの結果グリッドが表示され、画面上部に「📌 表示中: セッション名（件数）」が表示される

### セッションフォルダ名の見方

| 形式 | 意味 |
|------|------|
| `S1_20260506_01_バフ` | STEP 1、2026/05/06実行、当日1回目、キーワード「バフ」 |
| `S2_20260506_01` | STEP 2、2026/05/06実行、当日1回目 |
| `S3_20260506_01` | STEP 3、2026/05/06実行、当日1回目 |

## 12. よくある質問

### スクレイピングが突然止まった

**原因として多いもの：**
- AucFanがメンテナンス中またはサーバーエラー
- Chromeのセッションが切れた
- ネットワークが不安定

**対処法：**
1. 「■ 停止」ボタンを押して完全に停止させる
2. AucFanのページをリロードする
3. 再度「▶ スクレイピング開始」を押す（途中から再開される場合があります）

---

### 商品が少ししか表示されない

**考えられる原因：**
- 検索結果自体が少ない（AucFanで検索してページ数を確認）
- 価格フィルターで多くの商品が除外されている（デフォルト: 1,000〜3,000円）
- 同一商品グループが少ない（グループ化の閾値が高い）

**対処法：**
- `.env` ファイルの `MIN_PRICE` / `MAX_PRICE` を変更して価格範囲を広げる
- フィルターバーの「最小グループ数」を「1」に下げて全商品を表示する

---

### マスターリストへの追加条件は？

STEP 1完了時に、pHashグループ化後の**グループサイズが2件以上**のグループに含まれるセラーIDのみが追加されます。1件のみの商品（ユニーク品）のセラーは追加されません。

この閾値は `.env` の `MASTER_SELLER_MIN_GROUP_SIZE` で変更できます（デフォルト: 2）。

---

### AI判定が動かない

Gemini APIキーが設定されていない、または無効の可能性があります。

**確認方法：**
1. `~/Downloads/aucfan_tool/.env` ファイルをテキストエディタで開く
2. `GEMINI_API_KEY=` の後ろにAPIキーが入っているか確認
3. キーがない場合は [Google AI Studio](https://aistudio.google.com/app/apikey) で取得して設定

AI判定なしでも pHash による画像グループ化は動作します。

---

### 結果をiPhoneで確認したい

2つの方法があります。

**方法①：HTMLエクスポートをAirDropで送る**
1. フィルターバーの「📱 HTML書き出し」をクリック
2. ダウンロードしたHTMLファイルをAirDropでiPhoneに送る
3. iPhoneのSafariで開く（フィルター・検索機能付き）

**方法②：同じWi-Fiで管理画面を直接開く**
1. MacのIPアドレスを確認（ターミナルで `ipconfig getifaddr en0`）
2. iPhoneのSafariで `http://[MacのIPアドレス]:5001` を開く

---

### セラー分析で「seller_url: なし」と表示される

STEP 1のスクレイピング時にセラーURLが取得できなかった場合に表示されます。この場合はフォールバックURL（`https://aucfan.com/search1/?aucnm=セラーID`）が自動的に使用されるため、スクレイピング自体は問題なく実行できます。
